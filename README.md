# CS330-FinalProject

Working on this project had a positive impact on how I approach software design. The eight weeks I spent on this project reinforced the importance of planning and building features incrementally rather than trying to solve everything at once. I started by creating a basic scene with simple geometry and then gradually added textures, materials, lighting, camera controls, and finally a skybox. As the project grew, I organized the code into smaller rendering functions for different parts of the scene, which made it much easier to test changes and keep the project maintainable. This iterative approach also made debugging much more manageable since I could isolate problems to a specific object or system instead of searching through one large rendering function.

From a development perspective, this project gave me hands-on experience with concepts I had never used before, including the graphics pipeline, shaders, transformations, camera systems, and texture management. More importantly, it strengthened skills that apply well beyond graphics programming, such as modular design and debugging complex code. The experience showed me how computer science concepts like linear algebra and matrices come together in a real application. As I continue my education and work toward a career in software engineering, these skills will help me tackle larger and more complex projects, even if they aren't focused on computer graphics.
